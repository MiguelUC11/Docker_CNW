Principais códigos do Docker

Para o Docker de forma eficiente, é essencial conhecer seus principais comandos. Eles podem ser agrupados em categorias: imagens, containers, redes, volumes e gerenciamento geral.

1. Comandos de Gerenciamento de Imagens

  • docker pull <nome_da_imagem>:
    Baixa uma imagem do Docker Hub ou de outro repositório.
  • docker build -t <nome_da_imagem> <caminho_do_dockerfile>:
    Constrói uma imagem a partir de um Dockerfile.
  • docker images:
    Lista todas as imagens disponíveis localmente.
  • docker rmi <nome_da_imagem>:
    Remove uma imagem local.

2. Comandos de Containers

  • docker run -d --name <nome_do_container> <nome_da_imagem>:
    Inicia um novo container em segundo plano.
  • docker ps:
    Lista todos os containers em execução.
  • docker stop <nome_do_container>:
    Para um container em execução.
  • docker rm <nome_do_container>:
    Remove um container parado.
  • docker exec -it <nome_do_container> /bin/bash:
    Acessa o terminal de um container em execução.

3. Comandos de Redes

  • docker network create <nome_da_rede>:
    Cria uma nova rede Docker.
  • docker network ls:
    Lista todas as redes Docker.
  • docker network inspect <nome_da_rede>:
    Mostra detalhes sobre uma rede específica.
  • docker network rm <nome_da_rede>:
    Remove uma rede Docker.

4. Comandos de Volumes

  • docker volume create <nome_do_volume>:
    Cria um novo volume Docker.
  • docker volume ls:
    Lista todos os volumes Docker.
  • docker volume inspect <nome_do_volume>:
    Mostra detalhes sobre um volume específico.
  • docker volume rm <nome_do_volume>:
    Remove um volume Docker.

5. Comandos de Informações e Limpeza

  • docker info:
    Exibe informações detalhadas sobre a instalação do Docker.
  • docker stats:
    Mostra o uso de recursos em tempo real dos containers em execução.
  • docker system prune:
    Remove todos os containers parados, redes não utilizadas, imagens pendentes e caches de construção.

6. Docker Compose

  • docker-compose up -d:
    Inicia todos os serviços definidos em um arquivo docker-compose.yml em segundo plano.
  • docker-compose down:
    Para e remove todos os containers, redes e volumes definidos no docker-compose.yml.
  • docker-compose ps:
    Lista os containers gerenciados pelo Docker Compose.
