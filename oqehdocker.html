O que é o Docker

O Docker é uma plataforma de containerização que empacota aplicações e suas dependências em containers, garantindo que rodem de forma igual em qualquer ambiente.

O que são containers?

Como usar containers?

Containers são como pequenas “caixas” virtuais que isolam uma aplicação do resto do sistema operacional. Dentro deles ficam não só o código da aplicação, mas também todas as dependências necessárias, como bibliotecas e configurações. Isso garante que a aplicação rode de forma consistente em qualquer ambiente, sem problemas de compatibilidade. Além disso, os containers são leves e rápidos, pois compartilham o núcleo do sistema host.

Para usar um container, você pode configurar seus recursos de sistema como memória, CPU e armazenamento, suas variáveis de ambiente como senhas, APIs e URLs de banco de dados, suas dependências(pacotes) como softwares, bibliotecas e ferramentas, entre outras configurações. Depois disso, você empacota as aplicações ou parte delas nos containers e consegue rodá-las sem afetar os outros containers de sua aplicação, em caso de erros ou interferências. E caso você deseje alterar alguma coisa na aplicação, ou subir atualizações, você pode fazer tudo isso isoladamente, sem afetar os outros containers do sistema.

Quando aplicar containers?

Hoje em dia, quase todas as aplicações usam containerização. Porém, geralmente usa-se containers em casos de sistemas grandes, com diversos serviços e aplicações complexas, o que propicia a aplicação de sistemas isolados para que não haja interferência e disparidades entre cada parte do código. Costuma-se usar containers para testes e desenvolvimento, propiciando o mesmo ambiente para todos os desenvovlvedores, para divisão das funções do sistema em diversos microsserviços, e para tornar a aplicação mais escalável, permitindo que ela se adapte à diferentes situações com mais eficiência.
